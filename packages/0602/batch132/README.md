# bookscan - 二手书店 ISBN 扫码入库工具

摄像头或图片扫码 ISBN，自动查询 Open Library 补全书名作者，写入 SQLite 数据库。

## 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# macOS 还需要安装 zbar 系统库
brew install zbar

# 扫码入库
python3 bookscan.py scan --camera          # 摄像头实时扫码
python3 bookscan.py scan --image book.jpg  # 从图片扫码
python3 bookscan.py scan 9780140328721     # 直接输入 ISBN
python3 bookscan.py scan --folder ./books  # 批量扫描文件夹
```

## 📖 条码识别原理 & 拍照技巧

这一节专门给门店小伙伴看～扫不出来的时候对照一下！

### 1. pyzbar 是怎么识别条码的？

```
        图片 → 灰度化 → 边缘检测 → 识别条形码 → 解码数字
          ↑
    越清晰、越正，越快！
```

**pyzbar 识别流程：**

1. **找条码**：扫描整张图，找一组黑白相间、宽窄不一的竖条
2. **解码条宽**：每个条/空的宽度对应一位数字（宽的 = 1，窄的 = 0）
3. **校验**：最后一位是校验位，算一下对不对
4. **输出**：返回识别到的字符串

**关键点：pyzbar 认的是「竖条」。** 如果条码是横的、倒的、斜的，它就认不出来——

| 角度 | 识别率 |
|---|---|
| 0°（正着） | ✅ 100% |
| 90° / 270°（竖的） | ❌ 0%（除非程序帮你转） |
| 180°（倒的） | ❌ 0% |
| 15° 倾斜 | ⚠️ 看运气 |
| 透视变形（斜拍） | ❌ 基本没戏 |

---

### 2. 为什么书脊斜拍要先做透视校正？

**问题：** 你站着低头拍桌上的书，条码是梯形的，不是矩形。

```
  斜拍出来的条码 → pyzbar 看了：这啥玩意儿？

    _______         ╱╲
   /      /        ╱  ╲
  /______/        ╱____╲

  正拍（矩形）    斜拍（梯形变形）
```

**原因：**

- 条码的条和空本来应该是**平行等宽**的
- 斜拍后透视变形，左边的条变窄、右边的条变宽，比例全乱了
- pyzbar 按"等宽"去解码，算出来的数字全错

**怎么拍才对？**

✅ **手机正对着条码拍**，让条码在画面里是方方正正的矩形

❌ 不要斜着拍、从上往下拍、从侧面拍

---

### 3. 扫不出怎么办？按这个顺序试：

| 步骤 | 操作 | 说明 |
|---|---|---|
| ① | **把条码摆正** | 条码的竖条要和手机边缘平行，别歪 |
| ② | **离近一点** | 条码占满画面 1/3 大小最好，太小人眼都看不清 |
| ③ | **擦一下镜头** | 指纹、油印、灰尘都会糊 |
| ④ | **开闪光灯** | 光线暗的时候开闪光灯，注意别反光 |
| ⑤ | **换个角度** | 别斜拍！手机端平了对准条码 |
| ⑥ | **手动输入** | 实在扫不出来就手输 ISBN 号吧 |

```
  扫不出 → 摆摆正 → 凑近点 → 擦擦镜头 → 开灯 → 还是不行 → 手输
                                 ↓
                             5 秒解决 80% 的问题
```

**常见坑：**

- ❌ 书脊弯曲的平装书：条码被折成弧形了，用手压平再拍
- ❌ 磨砂封面：反光！换个角度避开反光
- ❌ 库存标签贴在条码上：把标签撕了再扫
- ❌ 条码在书脊侧面：把书翻开一点让条码正对镜头

## 其他命令

```bash
# 预览不入库
python3 bookscan.py scan 9780140328721 --dry-run

# 列出所有图书
python3 bookscan.py list

# 导出 CSV 盘点
python3 bookscan.py export -o 库存表.csv

# 录入价格和货架号
python3 bookscan.py scan 9780140328721 --cost 8 --price 18 --shelf A-03-12
```
